iOS 相关笔记

程序优化相关方案

  1. 通知中心: 建立通知中心, 完成全文件解耦

  2. 链式编程: 通过链式编程库进行属性快速设置, 保证代码优雅

  3. CSS 模式: 仿照 CSS 文件进行 iOS 组件样式设置

    方案构想

    UI 组件快速设置属性方案: & 各种组件的属性大概能分为两种, 一种是 Frame 必须手动设置, 一种是属性, 这个统一度比较高 & 设置内部可以通过键值对检查, 防止错误赋值 1.设置控件的 StyleSheet 属性, 通过枚举值进行控件设置, 枚举字符串要见名知意 2.设置控件的 Style, ID, Class 属性, 通过 id 和 class 进行赋值 3.总体目标: 基于控件属性的高度相似原理, 写一遍属性, 用到各种地方, 主要问题在于 1.如何调用, 减少学习成本且快速 2.如何组织实现过程, 保证高效以及高度复用性 整理: 以上问题的回答基础可能需要基于对现有各种控件属性的总结, 分为布局型, 属性设置型, 属性设置型中又能区分为通用型, 特殊型( 例如文字大小等 ), 其实这一套东西放在任何地方都可以通用, h5, ios, android, java, 其实这些语言的 GUI 界面的所有元素基本是通用的, 甚至可以基于这个原理, 开发出一套通用的属性映射表, 通过脚本甚至可以在各个平台进行通用性设置

  4. 单例享元模式: 创建唯一的享元单例, 管理项目所有可能通用的单例, 一来不需要每个工具类都要创建单例, 方便管理, 二来方便工具类的查找和使用

    方案构想

    iOS: 应该创建一个享元池, 本身是单例, 内部管理多个本来应该是单例, 但是现在不需要写成单例的单例对象, 一般来说, 这些对象的业务目标都比较单一和独立, SinglePool.object 形式, 便于管理, 点进去就能知道本地有哪些单例业务类, 具体的 object 实现采用懒加载形式进行加载
